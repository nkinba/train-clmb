# UI.md (모바일 웹앱 UI 가이드)

PRD §6 사용 시나리오를 모바일 PWA에서 만족시키기 위한 UI 결정 기록.
와이어프레임은 추후 디자인 단계에서 상세화하되, 본 문서는 **결정과 그 근거**를 박아둔다.

## 1. 사용 컨텍스트 (UI 결정의 출발점)

| 조건 | UI 함의 |
|------|---------|
| 한 손 조작 (다른 손은 홀드/분필주머니) | 핵심 버튼은 하단 영역 (엄지 도달) |
| 손가락에 분필/땀 | 탭 영역 최소 48dp, 스와이프/롱프레스 의존 금지 |
| 행보드 룸 조명 가변 | 다크 모드 기본, 시스템 추종 |
| 세션 중 5–10회 빠른 입력 | 화면 깊이 ≤ 2, 모달 중첩 금지 |
| 폰 거치 + 인터벌 진행 | Wake Lock으로 화면 유지 |
| Wi-Fi 불안정 | 오프라인 인디케이터 + 큐 길이 표시 |

## 2. 글로벌 패턴

* **하단 탭 네비게이션** (4탭): `오늘 / 기록 / 분석 / 설정`. 햄버거 메뉴 금지.
* **세로 고정** — 행보드 룸 거치 환경 대응.
* **다크 모드 기본** — 시스템 설정 추종.
* **마지막 값 자동 채움** — 그립, 홀드 깊이, +kg는 직전 입력값 디폴트.
* **+/− 스테퍼**가 슬라이더보다 우선 — 정밀 입력에 유리.
* **오프라인 인디케이터** — 상단 얇은 띠로 표시 + 큐 길이.

## 3. 화면별 결정

### 3.1. 홈 (오늘 탭)
* 상단: 오늘 세션 시작 버튼 (없을 때) / 진행 중 세션 카드 (있을 때).
* 하단: 최근 7일 요약 (행보드 세션 수, 등반 시도, 통증 평균).
* 모듈 선택 카드 (행보드 / 등반 / 보조) — 큰 타일 형태.

```
┌────────────────────────┐
│ 오프라인 · 큐 2개      │  ← 상단 인디케이터
├────────────────────────┤
│  [ + 세션 시작 ]       │  ← 한 손 도달
│                        │
│  최근 7일               │
│  행보드 4세션 · 등반 2  │
│  통증 평균 0.5         │
│                        │
│  ┌──────┐ ┌──────┐    │
│  │ 행보드│ │ 등반 │    │
│  └──────┘ └──────┘    │
│  ┌──────┐ ┌──────┐    │
│  │ 보조 │ │캠퍼스│    │
│  └──────┘ └──────┘    │
├────────────────────────┤
│ 오늘  기록  분석  설정 │  ← 하단 탭
└────────────────────────┘
```

### 3.2. 행보드 인터벌 타이머
* **풀스크린**. 진입 시 Wake Lock 자동 요청.
* 상단: 현재 세트 / 총 세트 (예: `3 / 5`), 그립/홀드/+kg 요약.
* 중앙: **거대한 숫자** (남은 초), 색상으로 단계 구분 (매달리기=빨강, 휴식=초록, 카운트다운 3s=노랑).
* 하단: 일시정지 / 종료 버튼 (큰 영역).
* 신호: 3초 전 비프(Web Audio) + 진동(Vibration API). 매달리기 종료 시 길게 진동.

```
┌────────────────────────┐
│ 세트 3 / 5             │
│ 하프 크림프 · 18mm·+5kg│
├────────────────────────┤
│                        │
│        0:08            │  ← 풀스크린 거대 숫자
│      매달리기           │
│                        │
├────────────────────────┤
│ [ 일시정지 ] [ 종료 ]  │
└────────────────────────┘
```

### 3.3. 세트 결과 입력 (매달리기 종료 직후)
* 화면 반반으로 **성공 / 실패** 두 버튼.
* 실패 길게 누르면 실제 버틴 초 입력 다이얼로그 (휠 대신 +/− 스테퍼).
* RPE 입력은 휴식 시간 중 카드 형태로 한 번만 (세트 5개 후 종합 입력 옵션).

```
┌────────────────────────┐
│ 세트 3 결과            │
├────────────────────────┤
│                        │
│   ✓ 성공                │  ← 화면 절반
│                        │
├────────────────────────┤
│                        │
│   ✗ 실패 (길게: 초)    │  ← 화면 절반
│                        │
└────────────────────────┘
```

### 3.4. 등반 빠른 입력 (볼더링)
* 프로젝트 이름 자동완성 (직전 세션 동일 이름 우선).
* `시도 +1` / `완등` 두 버튼이 큰 영역.
* RPE는 세션 종료 시 일괄 입력 옵션.

```
┌────────────────────────┐
│ 프로젝트: V7 노란 슬랩 │  ← 자동 채움
│ 누적 시도 4 · 완등 0   │
├────────────────────────┤
│  ┌──────────────────┐  │
│  │  + 시도          │  │  ← 큰 버튼
│  └──────────────────┘  │
│  ┌──────────────────┐  │
│  │  ✓ 완등!         │  │
│  └──────────────────┘  │
│                        │
│ [ 노트 추가 ]          │
└────────────────────────┘
```

### 3.5. 분석 (분석 탭)
* PRD §7의 차트 (그립별 +kg 라인, 주간 볼륨 바, 통증 캘린더, 프로젝트 타임라인).
* 모바일이므로 차트는 1개씩 세로 스크롤. 핀치줌 대신 기간 필터 칩 (1M / 3M / 6M / 1Y).

## 4. 기술 결정 (UI 관련)

| 항목 | 결정 |
|------|------|
| PWA 구성 | Workbox로 Service Worker 수동 구성 (`next-pwa`는 Next 15+ 호환성 시점 확인) |
| 오프라인 큐 | IndexedDB에 입력 큐 저장, 온라인 복귀 시 PocketBase로 flush |
| 화면 유지 | Wake Lock API (`navigator.wakeLock`) — 타이머 화면 진입/이탈 시 요청/해제 |
| 오디오 | Web Audio API (정확한 스케줄링), `<audio>` 비권장 |
| 햅틱 | Vibration API (Android만 — iOS는 미지원, 비프로 대체) |
| 차트 | Recharts 또는 visx (정적 export 호환) |
| 폼 입력 | React Hook Form + Zod (스키마 일관성) |
| 스타일 | Tailwind CSS (ADR 3 명시) |

## 5. 접근성/안정성 체크리스트

* [ ] 모든 핵심 탭 영역 ≥ 48dp.
* [ ] 다크 모드 대비비 WCAG AA 충족.
* [ ] 오프라인 진입 후 모든 입력 화면 동작 (Service Worker 캐시).
* [ ] 타이머 진입 시 Wake Lock 활성 표시.
* [ ] iOS Safari에서 비프/진동 동작 확인 (Vibration 미지원이라도 비프는 동작).
* [ ] 세션 중 핵심 입력(매달리기 결과, 등반 시도)은 ≤ 2탭.
